1          Public Data Structure and API

Public Data Structure

Person : An object has below properties.   This data structure is used both in following data provider and PeoplePicker API.

displayName : The display name of the person.  

description : Any descriptive info of the person. It could be job title, email, etc.  

personId : A unique identifier of current person object.

 

ValidationError : An error object, which was used by PeoplePicker error mechanism.

errorName : The error name. In this version, the possible values include:

                   “MultipleMatch”: There’s multiple match results for current resolved item.

                   “MultipleEntry”: There’s multiple items in PeoplePicker while allowsMultipleSelection is false.

                   “NoMatch”: No search result for current input.

                   “ServerProblem”: The data provider can’t provide data due to server unavailable.

localizedErrorMessage : A default error message provided by PeoplePicker.

 

PeoplePicker control API

We provide a one-time initializer for PeoplePicker control in this version.

Office.Controls.PeoplePicker(container, dataProvider, options)

container : The container HTML element to host PeoplePicker control.

dataProvider : A data provider object to provide data for PeoplePicker. In most cases, it returns data asynchronously. See data provider below.

options : Optional. Object contains various options supported by PeoplePicker, includes:

allowMultipleSelections : bool. If it’s true, more than one person can be added to control. The default value is false.

startSearchCharLength : int. The minimal characters length to trigger search begin. The default value is 3. The minimal value is 1.

delaySearchInterval : int value in ms. The interval between one key down and sending searching request. The default value is 250ms.

enableCache : bool. If it’s true, the cache will be regarded as another searching data source, every selected person is added to cache. If it’s false, the cache won’t be used.  The default value is true.

inputHint : string. The tooltip to show in the searching box when it’s empty. The default value is "Type a name...".

showValidationErrors : bool. Whether to enable default PeoplePicker error mechanism. The default value is true.

options can also contain event handlers when certain event happens. All event handlers by default take PeoplePicker control as the parameter.

onAdded(control, person) : The event handler of adding a person to control. If the person is an unresolved person , only its display name will have value user typed.

onRemoved(control, person) : The event handler of removing a person to control. If the person is an unresolved person , only its display name will have value user typed.

onFocus : The event handler of when PeoplePicker gets focused.

onBlur : The event handler of when PeoplePicker gets blurred.

onError(control, validationError) : The error handler of when PeoplePicker has error.

 

PeoplePicker also has following methods for developer to use after initialization.

PeoplePicker methods

getErrorDisplayed() : Return the validationError PeoplePicker displays now when showValidationErrors is set to true.

getAddedPeople() : An array of person object. These are the items displayed inside people picker.   If the person is an unresolved person, only the property display name will have value user input.

clearCacheData() : Clear cache data when cache is enabled when initialize PeoplePicker and stores some selected people which you want to clear.

reset() : Reset the PeoplePicker to initial state.

remove(person) : Remove an added person in PeoplePicker. The person object could be retrieved from PeoplePicker. getAddedPerson ().

add(person, resolved) : Add a person in PeoplePicker.

         When person is string, then the value of resolved is ignored and PeoplePicker would call data provider to resolve the person.

         When person is a person object, then resolve is optional.

                   If resolved is undefined or false, PeoplePicker would call data provider to resolve the person.

                   If resolved is true, PeoplePicker would treat it as a resolved item and not call data provider.

 

Data provider

The data provider is an object which providers below function for PeoplePicker to call.

function searchAsync(keyword, callback)

keyword : string. The search string user inputs in PeoplePicker to trigger a new search process.

callback : function(error, data). Data provider must call this function to tell PeoplePicker a search is complete.

error : string. Contains error message if error happens.

data : An array of person object. Contains data if search is complete without any error.

person : An object must have below properties.

displayName : The display name of the person.

description : Any descriptive info of the person. It could be job title, email address, etc.

personId : A unique identifier of current person object.

 

 

2          AAD data provider

AAD data provider ships with PeoplePicker as a sample. Developers can extend this sample to meet their scenario to retrieve data from AAD. AAD data provider is composed of two parts, a client data provider object and server side implementation to do authentication and Graph API calls.

AadDataProvider client only provide one extra property maxResult.

AadDataProvider properties

maxResult : int. The maximum result count returned in one search.

AadDataProvider methods

searchAsync(keyword, callback) : Implements PeoplePicker desired behavior.

 

AadDataProvider client talks to server using one endpoint /api as illustructed below.

 

We also provide a convenient endpoint /login for user to login when load page. Developer can decide when to authenticate user.

AadDataProvider server-side endpoints

/login : Developer can call this endpoint to authenticate user with AAD whenever he/she wants.

/api?keyword=<keyword> : AadDataProvider client call this endpoint with keyword as query string. The server takes care of authentication and calls Graph API to return user information.
